Este documento detalla el funcionamiento del módulo de catálogo de productos. El sistema lista los productos disponibles junto con su marca y categoría, garantizando que el cliente final solo visualice los artículos que cuenten con stock disponible mayor a cero.

Prerrequisitos de Instalación y Base de Datos

Para que este módulo funcione de forma correcta, es obligatorio preparar la base de datos con las estructuras correspondientes. Se deben ejecutar los scripts SQL incluidos en el proyecto para crear las tablas de productos, marcas y catálogo. Asimismo, es necesario registrar el procedimiento almacenado denominado pa_list_catalog_products que se encuentra en los archivos de script. Este procedimiento es fundamental ya que realiza la unión de las tablas mediante operaciones JOIN y extrae la información unificada de la tienda en un solo bloque.

El sistema procesa la información a través de cuatro capas de software bien definidas:Primero, la Base de Datos MySQL procesa la consulta a través del procedimiento almacenado mencionado y entrega los registros crudos.Segundo, la capa de Repository en Spring Data JPA invoca dicho procedimiento y recibe las filas en forma de una lista de arreglos de objetos genéricos.Tercero, la capa de Service contiene toda la lógica de negocio. Esta capa recorre el resultado de la base de datos, analiza el inventario, descarta los productos agotados y redacta el mensaje final de respuesta.Cuarto, la capa de Controller expone el punto de acceso HTTP para que las aplicaciones cliente o interfaces web puedan consumir el recurso de manera segura.
